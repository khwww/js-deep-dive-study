# 44. REST API

REpresentational State Transfer는 HTTP의 장점을 최대한 활용할 수 있도록 만든 일종의 아키텍처이다.

## 44.1 REST API의 구성

| 구성 요소              | 내용                           | 표현 방법        |
| --------------------- | ----------------------------- | --------------- |
| 자원(resource)         | 자원                           | URI(엔드포인트)  |
| 행위(verb)             | 자원에 대한 행위                | HTTP 요청 메서드 |
| 표현(respresentations) | 자원에 대한 행위의 구체적 내용   | 페이로드         |

## 44.2 REST API 설계 원칙

- URI는 리소스를 표현
- 행위에 대한 정의는 HTTP 요청 메서드로 한다
